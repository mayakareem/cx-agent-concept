The Concept

Problem

Today, journey mapping and CX measurement are disconnected workflows. Teams spend weeks in workshops mapping journeys that never connect to live data. CX programs set up touchpoint surveys without a journey context. The result: static maps that gather dust and measurement programs that miss the full picture.

Solution: The CX Agent

A two-part AI agent that sits across JM and CX products. Part 1 builds the journey through an intelligent interview - asking the right questions to construct stages, touchpoints, channels, emotions, and pain points. Part 2 takes that journey and auto-configures the entire CX measurement program - survey design, distribution channels, integration setup, inclusion/exclusion rules, intercepts, and dashboards.

The journey map becomes the config file for the entire CX program.

Question Bank & Decision Logic

What industry or sector is this for?
Banking & Financial Services Telecom Retail & E-commerce Healthcare Government & Public Sector Hospitality Insurance Automotive Education Other (free text)
Decision: Selection loads an industry-specific journey template with pre-populated stages. E.g., Banking loads: Awareness - Application - Onboarding - Active Use - Service Request - Renewal/Loyalty. Telecom loads: Discovery - Purchase - Activation - Usage - Support - Upgrade/Churn.
What product, service, or experience is this journey about?
Free text input Examples shown based on industry
Output: Agent uses this to label the journey and customize touchpoint names. E.g., "Credit Card Application Journey" or "Broadband Installation Journey"
Who is the customer? Is this B2B or B2C? Can you describe the primary persona?
B2C Individual B2C Household B2B Decision Maker B2B End User B2B Buying Committee
Decision: B2B triggers additional questions about buying committee roles, account hierarchy, and multi-stakeholder touchpoints. B2C focuses on individual emotional journey. B2B also adjusts default measurement cadence (less frequent, more strategic).
Do you have an existing journey map or process flow? (Upload PDF, PPT, Excel, image, or describe it)
Upload file Paste text/description No, start from scratch
Decision - If uploaded: Agent parses the document using AI (leveraging the existing AI-Assist Import Map feature from JM). Extracts stages, touchpoints, channels, and any metrics mentioned. Presents parsed journey for confirmation before enriching.
Decision - If no existing map: Agent uses the industry template and begins guided questioning to build from scratch.
What triggers this journey? What causes a customer to enter it?
Customer-initiated (inquiry, search, visit) Event-driven (life event, contract expiry, need) Company-initiated (marketing, outreach, offer) Recurring (subscription renewal, periodic service)
Output: Defines the journey entry point and first touchpoint. Also determines if the journey is linear vs. cyclical.
Here are the stages I'd suggest based on your inputs. Can you confirm, add, remove, or reorder?
Agent presents a suggested stage list User drags/reorders User adds custom stages User removes irrelevant stages
Output: Confirmed stage sequence becomes the backbone of the journey. Each stage will be expanded with touchpoints next.
For [Stage Name]: What touchpoints or interactions happen here? What channels do customers use?
Website/Portal Mobile App Physical Branch/Store Call Center Email WhatsApp/Chat Social Media SMS Self-service Kiosk In-person Meeting Third-party (broker, agent)
Decision: Each channel selected generates a touchpoint node in the journey. Multi-channel stages create parallel paths. Agent also flags "channel switching" risks (e.g., started online, had to call - known friction pattern).
Where do handoffs happen? (Between departments, systems, or channels)
Agent highlights likely handoff points based on stage transitions User confirms and adds detail
Decision: Handoff points are automatically tagged as high-risk touchpoints. These get priority measurement and alert rules in Part 2.
What are the known pain points or failure modes? Where do customers complain, drop off, or churn?
Per stage - user flags issues Agent suggests common pain points for the industry
Output: Pain points are mapped onto the emotional curve and tagged for deeper measurement in Part 2.
What are the Moments of Truth - the interactions that make or break the relationship?
Agent suggests based on industry patterns User selects or adds custom MoTs
Output: MoTs get priority status in the journey map. In Part 2, these receive enhanced measurement (richer surveys, real-time alerts, executive dashboards).
Are there different customer personas who experience this journey differently?
Yes - define personas No - single journey
Decision - If yes: Agent asks for persona names and where the journey diverges. Creates persona overlay layers on the same journey (not separate journeys). Divergence points become segmentation triggers for measurement in Part 2.

Part 1 Output: Journey Map

The agent generates a complete journey in the JM tool containing: named stages in sequence, touchpoints per stage with channel labels, emotional curve (positive/negative trajectory), pain points and Moments of Truth tagged, handoff points flagged, persona overlays (if applicable), and industry benchmark annotations. The user reviews and refines - then triggers Part 2.

Per-Touchpoint Configuration Logic

How should we measure this touchpoint? (Agent auto-suggests based on touchpoint type)
Transaction Survey Relationship Survey Web/App Intercept Behavioral/Passive (no survey) Combined
Auto-suggestion logic: Post-purchase/post-service touchpoints default to Transaction Survey. Periodic check-ins default to Relationship Survey. Website/app browsing stages default to Intercept. System handoffs default to Behavioral/Passive. Moments of Truth get enhanced measurement (Transaction + Follow-up).
What system or platform owns this touchpoint? (Where does the interaction happen or get recorded?)
Salesforce HubSpot Zendesk/Freshdesk Custom CRM POS System Website (tracked) Mobile App Call Center (Genesys/Avaya/etc.) ERP Manual/Offline
Output: Determines integration method. Agent checks if QuestionPro has a native integration (Salesforce, HubSpot, Zendesk, etc.) and auto-configures. For others, recommends API/FTP/webhook approach with setup instructions.
How should we reach the customer for this touchpoint?
Email SMS WhatsApp In-app notification Web intercept (pop-up/slide-in) QR code (physical location) IVR (post-call) Embedded in process
Decision: Channel selection triggers format constraints. SMS/WhatsApp limits survey to 1-2 questions. Email allows 3-5 questions. Intercept allows 1-3. IVR limits to 2 with numeric responses. Agent also flags if the selected channel matches the touchpoint channel (e.g., measuring a WhatsApp interaction via WhatsApp survey = contextual; via email = delayed).
When should the survey go out after the interaction?
Immediately (real-time trigger) Delayed - 1 hour Delayed - 24 hours Delayed - 48 hours Delayed - 1 week Custom delay Batched (daily/weekly)
Decision: Agent recommends based on touchpoint type. Service interactions = immediate or 1hr. Purchase = 24-48hrs (after delivery/usage). Onboarding = 1 week. Relationship = scheduled cadence. Agent flags if timing conflicts with other touchpoint surveys (survey fatigue risk).
What rules should govern who gets surveyed?
Frequency cap (e.g., max 1 survey per 30/60/90 days) Sampling rate (e.g., 25% of interactions) Segment-based (VIP vs. standard) Exclude internal users Exclude test accounts Exclude recently surveyed Exclude recently complained Include only completed interactions Custom rule
Output: Agent generates the CX workflow rules config: inclusion criteria, exclusion list, quarantine period, and sampling logic. Flags potential conflicts (e.g., if two touchpoints could trigger surveys on the same day for the same customer).
Agent auto-generates the survey questions for this touchpoint. User reviews.
Output per touchpoint:
- Anchor metric: NPS, CSAT, or CES (agent recommends based on touchpoint type - CES for service/support, CSAT for transactional, NPS for relationship)
- 1-2 driver questions specific to the touchpoint (e.g., "How easy was it to complete your application?" for an onboarding touchpoint)
- 1 open-ended follow-up ("What could we improve?")
- Optional: attribute ratings if touchpoint involves multiple dimensions (e.g., branch visit = staff friendliness + wait time + resolution)

Dashboard Auto-Layout

The agent structures the dashboard to mirror the journey stages. Each stage gets a section with its touchpoint KPIs, so stakeholders can read the dashboard like reading the journey. Top-level view shows the full journey health. Drill-down shows per-touchpoint, per-segment, per-channel detail.

Closed-Loop Intelligence

Beyond simple alerts, the agent tracks resolution outcomes. If a detractor is followed up and their issue resolved, the agent measures whether the next touchpoint score improves. Over time, it learns which recovery actions actually work and recommends them automatically.

Additional Capabilities to Add

Sentiment Layer (Real-time)

Beyond survey responses, capture sentiment from open-ended text across all touchpoints. AI auto-categorizes themes, detects urgency, and overlays sentiment onto the journey map in real-time. The journey map becomes a live heatmap of customer emotion.

Enhances Part 2 & 3

Journey Evolution Tracker

Journeys change over time - new channels, new products, shifted behavior. The agent monitors touchpoint usage patterns and flags when the journey has materially changed. Suggests map updates and new measurement points automatically.

Post-launch Intelligence

Predictive Churn Signals

Correlate touchpoint scores across the journey to predict churn before it happens. E.g., if a customer scores low on onboarding AND support within 30 days, the agent flags them as high churn risk and triggers a retention workflow.

Advanced Analytics

Multi-Persona Journey Variants

Same journey, different experiences. The agent manages persona overlays - VIP vs. standard customers, different geographies, different segments - with tailored measurement at divergence points. Dashboard filters by persona.

Enhances Part 1

Competitive Benchmark Integration

Overlay industry benchmarks on each touchpoint score. "Your onboarding CES is 4.2 - the banking industry average is 3.8." Uses QuestionPro's existing benchmark data plus public NPS/CSAT databases.

Dashboard Enhancement

Cost-of-Experience Calculator

Attach financial impact to each touchpoint. "Every detractor at the service touchpoint costs $X in churn risk." Links CX scores to revenue data (if integrated) to quantify the ROI of improving specific touchpoints. Directly addresses the Gartner gap on "Showing Business Outcomes."

Revenue Linkage

Survey Fatigue Manager

Cross-touchpoint intelligence that prevents over-surveying. Tracks each customer's survey exposure across the entire journey and dynamically adjusts sampling, skips surveys, or switches to passive measurement when fatigue risk is high.

Rules Engine

Auto-RCA (Root Cause Analysis)

When a touchpoint score drops, the agent automatically cross-references other data: Did operational metrics change? Is there a segment pattern? Did a process change coincide? Surfaces probable root causes without manual investigation.

AI Analysis

EX-CX Linkage

Connect employee engagement data from the EX platform to CX touchpoint scores. "Branches where employee engagement dropped 10% saw NPS decline by 8 points." Requires both EX and CX licenses - powerful upsell opportunity.

Cross-product

Natural Language Journey Queries

"Why did NPS drop at the onboarding touchpoint last month?" The agent answers in natural language, pulling data from dashboards, cross-referencing open-ends, and suggesting actions. Think ChatGPT for your CX program.

AI Interface

Regulatory & Compliance Layer

Auto-configure RAA (Respondent Anonymity Assurance), GDPR consent flows, data retention policies, and opt-out management per touchpoint. Critical for MEA markets (PDPL in Saudi, DIFC Data Protection in UAE).

Compliance

Stakeholder Report Generator

Auto-generate weekly/monthly CX reports per journey stage, per persona, or per region. Tailored to audience - executive summary for leadership, detailed breakdown for operations, action items for frontline managers.

What Makes This Different

Competitors offer journey mapping OR CX measurement. Nobody offers an AI agent that builds the journey through conversation and then auto-configures the entire measurement program. The journey map stops being a workshop artifact and becomes the live configuration that drives the CX program. Every touchpoint on the map is a live data point. Every stage is a dashboard section. Every pain point is being measured.

The agent doesn't just map the journey - it operationalizes it.
